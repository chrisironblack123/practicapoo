Programacion Orientada a Objetos (19/10/25 a 26/10/25)

Preguntas conceptuales y de Aplicacion

1-¿Qué es la programación orientada a objetos (POO)?

es un modelo de programación que organiza el código en torno a objetos, que son entidades
que agrupan datos (atributos) y funcionalidades (métodos). En lugar de un enfoque secuencial,
la POO organiza el software como un conjunto de objetos que interactúan entre sí, similar a cómo
pensamos sobre el mundo real.

2-Define los conceptos de clase y objeto.

Una clase es una plantilla o modelo que define las propiedades (atributos) y el comportamiento
(métodos) que tendrán los objetos. Un objeto es una instancia concreta de una clase, es decir, una entidad real o abstracta que se
crea a partir de esa plantilla, con sus propios datos específicos.

3-Menciona los cuatro pilares de la POO.

Los cuatro pilares de la programación orientada a objetos (POO) son la abstracción, el encapsulamiento,
la herencia y el polimorfismo.

4- Da un ejemplo de una clase sencilla llamada Vehiculo con un método acelerar().

class Vehiculo : def_init_(self, marca) : self.marca = marca # propiedad de la marca self,velocidad = 0 # atributo de la velocidad, inicial en 0 def acelerar(self, incremento): """Aumenta la velocidad del vehiculo.""" self.velocidad += incremento print(f"El {self.marca} acelero a {self.velocidad}km/h.") # por ejemplo mi_carro = vehiculo("Nissan") mi_carro.acelerar(60) mi_carro.acelerar(30)

5-¿Qué es la herencia y cómo se aplica?

La herencia es un mecanismo fundamental de la programación orientada a objetos (POO)que permite que
una nueva clase (clase derivada) herede atributos y métodos de una clase existente (clase base), lo que
promueve la reutilización de código y la creación de jerarquías de clases. Se aplica al crear una clase
que hereda las características de otra, y luego se puede extender o modificar ese comportamiento para
crear clases más específicas.

6- Explica el concepto de encapsulamiento con un ejemplo.

El encapsulamiento es un concepto de la programación orientada a objetos (POO) que agrupa los datos
(atributos) y los métodos que operan sobre ellos en una sola unidad, como una clase, y restringe el acceso
directo a los datos. Esto protege la integridad de los datos al obligar a que se modifiquen a través de los
métodos (conocidos como getters y setters), lo que permite controlar cómo y cuándo se accede a ellos. Por ejemplo, en una clase Coche, los atributos velocidadActual se pueden mantener privados, y se usan métodos
públicos getVelocidad() y setVelocidad() para interactuar con ellos.

7-¿Qué es el polimorfismo?

El polimorfismo en la programación orientada a objetos es la capacidad de tratar objetos de diferentes clases
como si fueran de una misma superclase común. Esto permite que un solo método, con el mismo nombre, pueda responder
de manera diferente según el objeto que lo invoque, haciendo el código más flexible y reutilizable.

8-¿Por qué la POO mejora la mantenibilidad del código?

La Programación Orientada a Objetos (POO) mejora la mantenibilidad del código gracias a la modularidad, que aísla
los cambios dentro de clases independientes, y la encapsulación, que protege los datos internos y simplifica las
interacciones. Estos principios permiten realizar modificaciones en una parte del código sin afectar al resto, lo
que facilita la corrección de errores y la actualización de funcionalidades.
